> ⚠️ Projeto conceitual para fins educacionais. Nintendo, Mario e personagens relacionados são marcas registradas da Nintendo Co., Ltd.
 
---
 
## 🚀 Sobre o projeto
 
O **Super Mario Galaxy: O Filme** é uma landing page que busca simular a experiência de um site oficial de blockbuster, permitindo:
 
- Apresentar o filme com uma hero section impactante
- Exibir os personagens principais com animações de scroll
- Navegar por trailers oficiais em um carrossel interativo
- Acompanhar a contagem regressiva para a estreia nos cinemas
O projeto foi construído do zero — do design no **Figma** até o código final — com foco em animações avançadas e qualidade visual.
 
---
 
## 🎬 Seções da página
 
### 🏠 Hero
Tela de entrada com vídeos animados em canal alfa de Mario, Yoshi e um planeta 3D giratório, sobrepostos a um campo de estrelas gerado via Canvas API.
 
### 🎭 Personagens
Galeria com os 6 personagens do filme — Mario, Luigi, Peach, Rosalina, Yoshi e Bowser Jr. — com efeito de partículas no fundo e animações ativadas por scroll.
 
### 🎞️ Trailers
Carrossel com 4 trailers incorporados do YouTube, navegável por setas, dots e teclado, com carregamento lazy dos iframes.
 
### ⏱️ Estreia
Seção final com countdown em tempo real para **25 de Dezembro de 2026**, com efeito de nebulosa e galáxia animadas no fundo.

## 🛠️ Tecnologias utilizadas
 
### 🎨 Frontend
- HTML5 semântico e acessível
- CSS3 (variáveis, animações, responsividade)
- JavaScript Vanilla
### ✨ Animações
- **GSAP 3** — ScrollTrigger e ScrollSmoother
- **Particles.js** — efeito de partículas na seção de personagens
- Canvas API — starfield dinâmico
### 🎨 Design
- **Figma** — prototipagem completa antes do desenvolvimento
- Fonte: [Outfit](https://fonts.google.com/specimen/Outfit) (Google Fonts)
### 🚢 Deploy
- GitHub Pages

## 🧠 Contexto de aprendizado
 
Este projeto foi desenvolvido como estudo de animações web avançadas e experiências imersivas no front-end.
 
- O foco principal foi dominar o **GSAP** com ScrollTrigger e ScrollSmoother
- Explorar o uso de **vídeos com canal alfa** para composição visual
- Praticar o fluxo completo de **design → código → deploy**
- Implementar recursos de **acessibilidade** (ARIA roles, live regions, navegação por teclado)
> Esse projeto foi essencial para entender como animações, performance e design visual se conectam em uma experiência de alta qualidade.
 
---
 
## 📚 Principais aprendizados
 
- Animações de scroll com GSAP (ScrollTrigger + ScrollSmoother)
- Uso de vídeos `.webm` com canal alfa para composição sem fundo
- Geração de efeitos visuais dinâmicos com Canvas API
- Construção de componentes interativos acessíveis (carrossel, countdown, nav flutuante)
- Fluxo de design no Figma antes da implementação

## 📌 Considerações finais
 
Esse projeto representa minha evolução no front-end além do básico — saindo de páginas estáticas para experiências com animação e design de alto nível.
 
Mesmo sendo um projeto conceitual, foi fundamental para:

---
 
## 🤖 Desenvolvido com IA
 
Este projeto foi inteiramente desenvolvido com o auxílio de **Inteligência Artificial**, desde a concepção visual até a escrita do código.
 
- O design foi criado com suporte de IA no **Figma**
- O HTML, CSS e JavaScript foram gerados e refinados com IA
- As animações com **GSAP** foram estruturadas e ajustadas com auxílio de IA
- O processo envolveu prompts, revisões e iterações constantes
> Isso não diminui o projeto — pelo contrário, faz parte do aprendizado entender **como usar IA como ferramenta de desenvolvimento**, saber o que pedir, revisar o que é gerado e tomar decisões técnicas e visuais ao longo do processo.
